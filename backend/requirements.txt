# Sakura backend runtime dependencies.
#
# Pins are intentionally written as >=, <next-major ranges (rather than ==X.Y.Z)
# so the same file installs cleanly on Python 3.10 through 3.13+. Several of
# these packages only publish prebuilt wheels for a narrow band of Python
# versions per release, so exact pins (cryptography==42.0.5,
# psycopg2-binary==2.9.9, pydantic==2.5.0, ...) would force a from-source build
# on newer interpreters and fail on machines without a C toolchain.
#
# If you need 100% reproducible builds, run `pip freeze > requirements.lock.txt`
# after a successful install and ship that lock file instead. The unpinned
# ranges below are what the public setup.{sh,ps1} scripts use.

Werkzeug>=3.1,<4
# cryptography ships prebuilt wheels for a narrow Python window per release.
